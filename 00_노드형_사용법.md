# 노드형 프롬프트 — 사용법

노드로 구조 설계 → JSON 내보내기 → Claude가 읽고 노드별로 글/웹 생성.

메인 도구: **노드에디터.html** (단일 파일, 무설치, 더블클릭=브라우저에서 열림).

---

## 1. 워크플로우 (매번 이 4단계)

1. **설계** — `노드에디터.html` 열고 노드 배치 + 연결선 그림
2. **내보내기** — 상단 `Claude용 내보내기` → 전체 복사 (또는 nodes.json 다운로드)
3. **호출** — Claude에게 붙여넣기 (지시문 자동 포함됨)
4. **생성** — Claude가 연결선 순서대로 노드를 순회 → 산출물을 `out/`에 저장

수정 = 에디터에서 노드만 고치고 다시 내보내기. 브라우저 자동저장(localStorage)됨.

---

## 2. 에디터 조작법

| 동작 | 방법 |
|------|------|
| 노드 추가 | 상단 `+ 리서치 / + 글 / + 웹 / + 프로그램` |
| 노드 이동 | 노드 **헤더** 잡고 드래그 |
| 제목·지시 수정 | 노드 안 텍스트 클릭해서 바로 입력 |
| 연결 | 노드 **오른쪽 점**을 잡고 → 다른 노드 위에 놓기 |
| 노드 삭제 | 노드 우측 `x` |
| 연결선 삭제 | 선 **클릭**(빨강) → `Delete` |
| 저장 | 자동 (localStorage). `불러오기`로 JSON 복원 |

---

## 3. 노드 규칙 (색 = 산출물 타입)

| 색 | 타입 | 의미 | 예 |
|----|------|------|----|
| 노랑 | research | 리서치·입력 | 페르소나, 자료, 데이터 |
| 초록 | write | 글 작성 | 카피, 기획안, 스크립트 |
| 파랑 | web | 웹·HTML | 랜딩, 섹션, 이메일 |
| 보라 | program | 프로그램 | 파이썬, 자동화 |

**연결선(edge) = 순서·의존.** `A → B` 면 "A 결과를 B 입력으로".

---

## 4. 노드 안에 쓰는 형식 (지시서)

노드의 spec = Claude에게 주는 미니 브리프. 이 3줄이면 충분:

```
- 입력: (어떤 노드 결과를 받는지)
- 지시: 구체 스펙 (형식·톤·개수·버전)
- 산출: 파일명 or 형식 (out/xxx.md)
```

지시가 구체적일수록 결과 정확.
"카피 써줘"(X) → "헤드라인+서브+CTA, A/B/C 3버전"(O).

---

## 5. 내보내기 JSON 구조 (Claude가 읽는 것)

```json
{
  "version": 1,
  "nodes": [
    {"id":"n1","type":"research","title":"타겟 페르소나","spec":"...","x":120,"y":120}
  ],
  "edges": [ {"from":"n1","to":"n2"} ]
}
```

Claude는 edges를 위상 정렬해 순서대로 각 노드를 처리함.
